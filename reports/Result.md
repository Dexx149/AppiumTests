# 📊 Отчёт по автоматизированному тестированию мобильного приложения «FMH Android»

## 1. Описание проекта

Автоматизированное тестирование проведено для мобильного Android-приложения `ru.iteco.fmhAndroid` с использованием стека:
- **Appium** (UIAutomator2) — драйвер для мобильного UI
- **JUnit 5** — тестовый фреймворк
- **RestAssured** — для API-тестов
- **Selenium** — обёртка для работы с элементами
- **Page Object Model** — паттерн организации кода
- **Allure** — отчётность

## 3. Сравнение ручного и автоматизированного тестирования

| Параметр | Ручное тестирование | Автоматизированное тестирование |
|----------|---------------------|--------------------------------|
| **Время одного прогона (20 тестов)** | ~2–3 часа | ~5–10 минут |
| **Время на разработку** | 0 часов | ~60–80 часов |
| **Точность** | Зависит от внимательности QA | 100% воспроизводимость |
| **Регрессионное тестирование** | Дорогое и долгое | Запускается в CI за минуты |
| **Параллельный запуск** | Невозможен | Возможен на нескольких устройствах |

---


## 4. Выводы
Из 20 запланированных тестов **все 20 автоматизированы** (17 E2E + 3 API). Ручное тестирование того же объёма занимает ~2–3 часа на один прогон и не масштабируется, тогда как автоматизированный прогон выполняется за минуты и может запускаться неограниченное количество раз без дополнительных затрат.
