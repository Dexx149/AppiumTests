# Appium Tests


## 📖 Документация по тестам

Подробная документация находится в папке **`reports`**.
После прогона тестов автоматически генерируется HTML-отчет Allure. Готовый отчет находится по пути: **`build/reports/allure/index.html`** (просто откройте этот файл в любом браузере).

---

## ⚠️ Текущие ограничения

На текущем этапе тесты стабильно проходят при запуске **по отдельности**. Однако при пакетном запуске всего набора тестов возможны случайные падения некоторых тестов из-за неучтенных зависимостей состояний или таймингов. Эта проблема известна, и механизм для её решения уже запланирован (см. раздел *Дальнейшие планы*).

## 🚀 Инструкция по запуску

### Шаг 1. Подготовка мобильного приложения
1. Скачайте исходный код мобильного приложения по ссылке: [архив с приложением](https://drive.google.com/drive/u/1/folders/14Sl8CAiIzFqtyXx6BAmWVlbu3_cXXzH1).
2. Распакуйте проект в любую удобную директорию на вашем компьютере.

### Шаг 2. Настройка путей
1. Откройте корневую папку **проекта тестов**.
2. Найдите файл `local.properties`.
3. Укажите в нем абсолютный путь к папке с мобильным приложением, которую вы скачали на Шаге 1. 
Пример:
appiumApp=C:\\Diplom\\fmh


### Шаг 3. Проверка окружения
Убедитесь, что все **Требования к системе** (см. таблицу выше) выполнены:
* Установлены нужные версии JDK (17 и 21) и настроены переменные окружения (`JAVA_HOME`, `ANDROID_HOME`).
* Установлен Node.js и глобально установлен Appium (`npm install -g appium`).
* Установлен драйвер UiAutomator2 (`appium driver install uiautomator2`).
* В Android Studio скачаны необходимые SDK.

### Шаг 4. Запуск скрипта
1. Откройте **PowerShell** в корне проекта тестов.
2. Запустите автоматизированный скрипт:
   .\run_tests.ps1

Скрипт автоматически соберет APK, поднимет эмулятор, запустит тесты и по завершению корректно закроет все фоновые процессы.

## 🗺️ Дальнейшие планы на развитие проект

В будущем планируется расширить функционал, улучшить архитектуру фреймворка и настроить инфраструктуру:

### 🏗 Архитектура и Фреймворк
- Вынести генерацию тестовых данных в отдельный класс.
- Сделать переиспользуемый компонент для хедера (Header).
- Переделать `FilterPage`: при использовании фильтра возвращать предыдущую страницу (например, `NewsPage` или `NewsControlPanelPage`).

### 🧩 JUnit 5 Extensions и работа с Issue Tracker
- Реализовать Extension для динамического пропуска тестов, помеченных аннотацией с номером Issue, если соответствующий Issue ещё не закрыт.
- Написать баг-репорты в Issue.
- Написать тесты для сценариев, которые ранее не были покрыты из-за багов, и пометить их указанным выше Extension.
- Реализовать Extension для подготовки и очистки тестовых данных через API (создание/удаление новостей перед/после API-тестов, так как на UI эти новости почти не отобразить).

### 📈 Покрытие тестами (Test Coverage)
- Улучшить существующие API-тесты.
- Реализовать выбор введенной даты и времени(сейчас всегда выбираются текущие).
- Добавить тест для клика по системной кнопке "Назад".
- Добавить тесты для кнопки "Cancel" в `FilterPage` и `EditNewsPage`.

### ⚙️ CI/CD и Инфраструктура
- Настроить интеграцию с Jenkins.
- Настроить параллельный запуск тестов.
- Реализовать механизм автоматического перезапуска упавших тестов для минимизации влияния нестабильных (flaky) тестов при пакетном прогоне.
